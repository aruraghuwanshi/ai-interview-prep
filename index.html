Anthropic · the full loop

Safety-first frontier lab. The culture-fit round is the #1 reason strong engineers get rejected — technical skill alone won't close it.

Timeline1–4 months · 7–9 rounds
Coding screenCodeSignal/Colab · 4 escalating gates · Python
AI toolsBanned live (auto-DQ) · OK for prep
Comp (Levels.fyi)Senior ~$563K · Lead ~$785K

Interview loop

  1. 1
    Recruiter screen(s) — background, "why Anthropic?", comp anchoring. A 2nd recruiter sends safety/values docs to study.
  2. 2
    Technical screen (60–90 min) — shared Python env; one problem escalating across 4 gates. Finishing isn't required; clear reasoning advances you.
  3. 3
    Hiring manager screen — engineering judgment, past projects, sometimes a code review.
  4. 4
    Onsite Loop 1 — system design + coding + culture fit. Fail Loop 1 → Loop 2 is cancelled.
  5. 5
    Onsite Loop 2 — experiences & goals + technical project deep-dive (prepare a presentation).

Coding practical, not LeetCode

  • In-memory database, progressively complex: set/get/delete → TTL → transactions (BEGIN/COMMIT/ROLLBACK) → prefix scan
  • LRU cache → add serialization/persistence → make it thread-safe (global lock passes)
  • Multi-threaded web crawler → filtered dictionary of crawled data
  • Longest-running function from stack-trace samples (+ infinite-loop edge case)
  • Read & eliminate duplicate files (dedup)
  • Justify a list of words within a given width
  • Serialize / deserialize a list of strings
  • Core business logic for a banking app (multiple transaction types)
  • Image-processing pipeline using unfamiliar libraries

Signal: your round-1 design should anticipate the constraints they haven't told you yet — decouple state from policy, leave clean hooks.

System design LLM infra + numbers

  • Design a batch-inferencing API (single GPU, ≤100/batch, users wait synchronously — maximize utilization)
  • Batch queries to optimize GPU usage → sharding, hotspots, geo-scaling follow-ups
  • Token-generation service handling up to 100,000 req/s
  • Distribute a 10GB file from one bandwidth-constrained source to thousands of hosts (P2P / BitTorrent-class)
  • Design a file cache system
  • Design an agentic AI system that autonomously adapts to new tasks

Name the tradeoffs: dynamic batching window vs P99, padding waste (sequence bucketing), KV-cache reuse, continuous batching (vLLM). For agents: least-privilege tools, human-in-loop before irreversible actions, capability evals, hard refusals.

Culture / values highest fail rate

  • Tell me about a time you had a moral conflict with the work. Who did you talk to? What changed your mind?
  • What concerns do you have with Anthropic's mission or direction?
  • How do you balance delivery speed with security?
  • A time you received negative feedback — how did you handle it?
  • A time you disagreed with your manager and later changed your mind.
  • Ethical risks of deploying agentic AI in high-stakes environments.
  • Why Anthropic specifically — not OpenAI, not DeepMind?
  • A time you built something that went against your values.
  • Your honest critique of Anthropic's direction.
  • A time the business problem wasn't clearly defined — how did you handle it?
  • A time you made a bold and difficult decision.
  • A conflict with someone: resolution + what you learned.

Interviewers push 3–4 follow-ups deep ("interrogative"). Shallow, sanitized stories fail — name real stakes, admit where you were wrong, hold complexity without collapsing to a tidy narrative.

ML / safety ML-adjacent roles

  • Handle hallucinations in a deployed model (layer training → inference/RAG → product → evals)
  • How does Constitutional AI change the RLHF loop? Tradeoffs?
  • Design an eval suite for a frontier model's capability & safety
  • A mechanistic-interpretability direction you'd pursue + success criteria
  • Model exposes training data — what next? Adjust model creativity? Define hallucinations?

The 4-pillar rubric what they score

  • Mission alignment + safety depth — engage with RSP/ASL, Constitutional AI, not "AI bias is bad"
  • Technical judgment under ambiguity — anticipate round-N; explicit tradeoffs, unprompted
  • Anthropic-research literacy — read the actual papers/blogs, not just the brand names
  • Collaboration + ego-low communication — credit others, explore feedback vs. argue it
Anthropic prep sprint: read the Responsible Scaling Policy v3 and Constitutional AI; know the ASL-3 activation (May 2025); form a real opinion on "why Anthropic not OpenAI/DeepMind"; drill escalating problems in Python AI-off; write STAR stories with an explicit safety layer.

OpenAI · the full loop

High coding bar, ML-systems-flavored design, and a mission round that screens for genuine alignment. Roles are often titled Member of Technical Staff (MTS) — same process.

Timeline2–8 weeks · 4–6 final rounds
Coding screenCoderPad · gate format · pass ≥2 · Python
AI toolsOff live · OK on take-home
Comp (Levels.fyi)L5 ~$440K–$580K + equity

Interview loop

  1. 1
    Resume + recruiter call — background, "why OpenAI?", team matching.
  2. 2
    Hiring manager screen — past systems, tradeoffs, light technical.
  3. 3
    Skills assessment — practical coding (CoderPad/HackerRank), sometimes a take-home / pair-coding or a design.
  4. 4
    Final loop (4–6 rounds) — coding ×1–2, system design, + at L5+: code refactor and a project deep-dive.
  5. 5
    Behavioral / mission — genuine perspective on AGI safety & beneficial AI. Then decision + offer.

Coding production-quality

  • LRU cache (get/put + eviction)
  • Versioned key-value store: get(key, timestamp) (time-travel reads)
  • KV store of lists of tuples → add sorting + merging across entries
  • Resumable iterator: next()/pause()/resume() from exact position
  • Implement cd path resolution (. / ..) → follow-up: symlink map
  • Excel-style formula evaluator (=A1+B2) → detect circular refs
  • Dependency version checker across constraints
  • GPU-usage + pricing → remaining credit balance
  • Concurrent web crawler (worker threads, dedup, failures)
  • Fix a race condition; pick + justify the primitive
  • Debug an O(n²) log parser → improve complexity, refactor, test
  • Alien dictionary; number of islands; topological sort

Bar: clean structure, intent-revealing names, explicit edge cases, and tests even when unasked. Brute force first, then iterate; narrate tradeoffs.

System design ML infra + scale

  • Serve ChatGPT to hundreds of millions of weekly users
  • Token-by-token streaming API (SSE vs WebSocket; worker dies mid-stream → resume)
  • Webhook delivery system: retries, backoff, DLQ, schema, idempotency
  • Rate limiter for a public API (per-org, back-pressure)
  • API for an in-memory database (WAL / MVCC)
  • Fault-tolerant web crawler at 10M req/s
  • GPU scheduling across competing workloads
  • Vector DB for billions of embeddings
  • Real-time NSFW detection on outputs (latency + feedback loop)
  • LLM enterprise search with role-based access control

Anchor on numbers: QPS, p95/p99, $/token. Talk continuous batching, KV-cache memory, prefill vs decode. Hand-waving "we'd use Redis" without why/QPS grades as junior.

Behavioral / mission SPSIL framing

  • Why do you want to work at OpenAI? (most repeated — nail it)
  • A project/accomplishment you're most proud of.
  • A large technical initiative you led.
  • A particularly complex technical problem — your approach.
  • A difficult technical trade-off decision.
  • A time you influenced a decision without authority.
  • A mistake you made — and the takeaway.
  • Managing multiple conflicting priorities.
  • A cross-functional disagreement — how you resolved it.

Mission alignment is weighted heavily — a rehearsed "I'm excited by AI" answer is a red flag. Read the OpenAI Charter and form real views.

Take-home / work trial AI allowed

  • Self-contained build (e.g., webhook delivery with retries) delivered async
  • Graded on production quality, tests, and documented trade-offs — not just features
  • No interviewer to bounce off → state and document your assumptions clearly

Round tips how to pass

  • Clarify scope + edge cases before writing code
  • Even after a correct answer, expect follow-ups — treat it as a conversation
  • Needing multiple substantive hints counts as a fail — show independence
  • Know your past projects deeply (architecture, what you'd change)
OpenAI prep sprint: read the OpenAI Charter + research blog; drill practical coding in a plain editor (no autocomplete) with tests; do design reps that force explicit numbers and streaming failure modes; simulate one 48h-style take-home end-to-end.
